# KidQuest — context for resuming work

> Read this first when picking the project back up. Status markers: 🟢 done · 🟡 in progress · 🔴 todo.

## What this is
Adaptive educational turn-based RPG (Heroes of M&M style). Daily content generated by a local
LLM, adapted per learner profile. Pure-TS game engine (Phaser only as a view), FastAPI server,
on-demand content pipeline. Full design in [ARCHITECTURE.md](ARCHITECTURE.md); plan-of-record in
`ROADMAP.md`.

## Ground rules (non-negotiable)
- **TDD always**: failing test first → minimum to pass → refactor green. Regression test before
  every bug fix.
- **English** for all code, comments, docs, commits. **Game content** is bilingual ES/EN.
- **Content safety (children)**: image generation uses an allowlist of family-safe pixel-art
  LoRAs; the NSFW LoRA `cris_v1*.safetensors` is BANNED. Generated text is safety-filtered.
- **Privacy**: real player profiles and generated content are gitignored; only generic example
  profiles ship.
- **REQ-CLI-01**: all game logic in pure TS under `client/src/engine/`, zero Phaser imports,
  green in Vitest before any scene wiring.
