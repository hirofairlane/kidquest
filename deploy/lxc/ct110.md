# Provisioning CT 110 (`kidquest`) on zeratul

KidQuest is served 24/7 from a Proxmox LXC on host `zeratul` (192.168.1.122). The
container only **serves pre-generated content** — no GPU, no inference. The daily
content batch runs separately on the GPU box (`blackwell`) and writes to the NAS
content store, which this CT mounts read-only.

> CT id **110** (109 is reserved for the CI runner; in use: 100,101,103,104,105,
> 106,107,108,200). Run these from `zeratul` (or via the CT104 bastion). This is a
> production change — execute deliberately.

## 1. Create the container

```bash
# Debian 12 unprivileged LXC, on the main node (always-on)
pct create 110 local:vztmpl/debian-12-standard_*_amd64.tar.zst \
  --hostname kidquest --cores 2 --memory 2048 --swap 512 \
  --rootfs local-zfs:8 \
  --net0 name=eth0,bridge=vmbr0,ip=dhcp \
  --features nesting=1 --unprivileged 1 --onboot 1
pct start 110
```

`nesting=1` is required to run Docker inside the LXC.

## 2. Mount the NAS content store (read-only)

The content store lives on the NAS (`/mnt/18T/MIO/Proyectos/KidQuest/content-store`,
exported by the plex CT / NAS). Bind-mount it into the CT:

```bash
# On zeratul: expose the NAS path into CT 110
pct set 110 -mp0 /mnt/18T/MIO/Proyectos/KidQuest/content-store,mp=/srv/kidquest/content-store,ro=1
```

(Or mount via NFS/SMB inside the CT if the NAS path is not local to the host.)

## 3. Install Docker + Compose in the CT

```bash
pct exec 110 -- bash -lc '
  apt-get update && apt-get install -y ca-certificates curl git &&
  install -m0755 -d /etc/apt/keyrings &&
  curl -fsSL https://download.docker.com/linux/debian/gpg -o /etc/apt/keyrings/docker.asc &&
  echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/debian bookworm stable" > /etc/apt/sources.list.d/docker.list &&
  apt-get update && apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
'
```

## 4. Clone and run

```bash
pct exec 110 -- bash -lc '
  git clone https://github.com/hirofairlane/kidquest /srv/kidquest/app &&
  cd /srv/kidquest/app/deploy &&
  KIDQUEST_CONTENT_DIR=/srv/kidquest/content-store docker compose up -d --build
'
```

The game is then served at `http://<ct110-ip>:8080` (add a DNS alias on the
OpenWrt router, e.g. `kidquest.lan`, the same way `homeassistant.local` is set).

## 5. Verify

```bash
pct exec 110 -- bash -lc 'curl -s http://localhost:8080/health'   # via nginx → api
# {"ok":true,"schema_version":"1"}
```

## Update

```bash
pct exec 110 -- bash -lc 'cd /srv/kidquest/app && git pull && cd deploy &&
  KIDQUEST_CONTENT_DIR=/srv/kidquest/content-store docker compose up -d --build'
```
